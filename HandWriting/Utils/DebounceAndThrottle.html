    手写防抖和节流

    # 防抖

    核心： 一段连续的操作结束后再执行回调，而不是每次操作都执行一次回调

    有时候，用户在搜索框中输入的时候，需要执行一个函数 fn 动态改变推荐的搜索关键词。如果用户每次输入都触发这个函数，那么它的执行频率就太高了，很影响性能。而对 fn 进行防抖处理之后，用户输入后的 n 秒内如果再次进行输入，则重新计时，直到用户输入后的 n 秒内不再输入，才调用 fn。

    实现代码如下：

    function debounce(fn,time){
    // 共享同一个 timer
    let timer = null
    return function(...args){
        clearTimeout(timer)
        timer = setTimeout(() => {
            fn.apply(this,args)
        },time)
    }
}
// 测试
const input = document.querySelector('input')
const fn = () => {
    console.log(this)
}
input.addEventListener('input',debounce(fn))

    通常情况下，给 DOM 添加事件监听的时候，回调函数中的 this 指向的是 DOM 本身，也就是说，fn 中的 this 指向的是 input。为了保持 fn 中的 this 上下文，防止它指向 window 对象，必须用 apply 给它绑定 this。这里的 this 位于箭头函数中，所以继承了父级词法作用域的 this，而父级刚好就是事件监听的回调函数，对它来说，this 指向的就是 input。

    # 应用场景

    • 搜索框动态推荐搜索关键词，避免每次输入都进行请求
    • 窗口大小 resize，避免每次调整都进行渲染

    # 节流

    **核心：**确保一段时间内只执行一次回调

    一种实现思路是利用时间戳，假设我们想要确保 4 秒内只执行一次回调，那么可以比较此次执行距离上次执行是否已经过了 4 秒，是则再次执行，否则不执行。实现代码如下：

    function throttle(fn,time){
    let pre = 0
    return function(...args){
        if(Data.now() - pre > time){
            fn.apply(this,args)
            pre = Date.now()
        }
    }
}

    另一种思路是利用定时器。我们可以做到 4 秒后再清空 timer，也就是说 4 秒内 timer 不会清空，同时让回调只在 timer 清空的前提下执行，就可以确保 4 秒内只执行一次回调了。代码如下：

    function throttle(fn,time){
    let timer = null
    return function(...args){
        if(!timer){
            timer = setTimeout(() => {
                timer = null
                fn.apply(this,args)
            },time)
        }
    }
}

    # 应用场景

    • 表单提交，限制 n 秒内只能提交一次，避免重复提交
