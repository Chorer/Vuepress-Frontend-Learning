    数据类型介绍

    vuePress-theme-reco    2021

    数据类型介绍

    • 基本数据（原始数据）类型有 6 种：Null、Undefined、String、Number、Boolean、Symbol

    • 引用数据类型有 2 种： Object（包含了对象字面量、包装类型、数组、Date、Error、RegExp、Math、Set、Map 等）、Function

    下面是部分数据类型的注意要点。

    # Number 类型

    # 关于 NaN

    NaN 是一种特殊的数字，本身也属于 number 类型，它的含义是“非数值”。涉及 NaN 的算术操作都返回 NaN，并且 NaN 不等于任何东西，甚至不等于 NaN（因为世界上的“非数值”很多，两个“非数值”可以不是同一个“非数值”）。如何检测 NaN 呢？有两个方法。

    isNaN() 会进行类型转换，看参数是否可以转化为数值，任何不能转化为数值的参数都被认为是“非数值”，并返回 true

    isNaN(10)  // false
isNaN("10")   // false
isNaN("blue")   // true
isNaN(NaN)      // true

    如果传进来的是对象，首先会调用 valueOf() 看是否可以转化为数值，如果不行，再看 toString() 是否可以转化为数值，如果也不行，则返回 true。

    Number.isNaN 是 isNaN 的加强版，它实际上更符合我们的直觉，所做的事情很简单，就是判断参数是不是 NaN 这个东西，是就返回 true

    Number.isNaN('blue')     // false   因为传进去的是字符串，不是 NaN
Number.isNaN(NaN)        // true    因为传进去的确实就是 NaN
Number.isNaN('a'-'b')    // true    因为传进去的 'a'-'b' 的运算结果确实就是 NaN

    # 关于转型函数

    Number() 可以将参数转化为数值：

    Number('12')         // 12
Number('12.0')       // 12    会移除小数点后的 0
Number('    12')     // 12    带有空格，不影响
Number('12abc')      // NaN   无法转化为数字  
Number('abcde')      // NaN   无法转化为数字 
Number(null)         // 0
Number(undefined)    // NaN   注意不是返回 0
Number("")           // 0
Number([])           // 0     传入对象，则先调用 valueOf，发现返回了自身，于是接着调用 toString，返回""，因此转化为 0
Number({})           // NaN   传入对象，则先调用 valueOf，发现返回了自身，于是接着调用 toString，返回"[object Object]"，无法被转化为数字

    parseFloat() 可以将参数转化为浮点数：

    parseFloat('12')       // 12    不会添加小数点
parseFloat('12.0')     // 12    会移除小数点后的 0
parseFloat(' 12')      // 12    带有空格，不影响
parseFloat('12abc')    // 12    截取数字部分
parseFloat('abc')      // NaN   无法转化为数字

    PS：对于布尔值、对象、数组、undefined、null，空字符串，均返回 NaN

    parseInt() 可以将参数转化为整数：

    parseInt('12.6')            // 12 
parseInt('12.0')            // 12
parseInt(' 12')             // 12   带有空格，不影响 
parseInt('12abc')           // 12   截取数字部分
parseInt('abc')             // NaN  无法转化为数字

    PS：对于布尔值、对象、数组、undefined、null，空字符串，均返回 NaN

    总结一下，三个转型函数都可以无视参数前面的空格，Number 会尽量将整个参数转化为数字，而 parseInt 和 parseFloat 则更多是解析参数的数字部分，且比较严格，很多值无法转化为数字

    # Symbol 类型

    # 用途一

    Symbol 的第一个用法是创建唯一的记号，通常作为对象的属性名，可以防止属性冲突的问题。几个要点：

    1. 即使传入相同的参数，也是不同的 Symbol 实例。Symbol('a') != Symbol('a')

    2. Symbol 不存在字面量形式，创建 Symbol 实例的方法只有两种：一种是 Symbol('a')（注意不能使用 new）；另一种是 Symbol.for('a')，这种方式创建的 Symbol 实例会放在全局符号注册表中，下次再调用 Symbol.for('a') 的时候会直接重用之前已经创建好的 Symbol 实例 —— 意思是 Symbol.for('a') === Symbol.for('a')。

    # 用途二

    第二个用法是提供了一些内置的 Symbol 属性名，比如 Symbol.hasInstance、Symbol.toPrimitive 等，从而暴露内部行为，方便开发者进行自定义。

    1. Symbol.hasInstance：这个方法挂载在 Function 原型上，也即 Function.prototype[Symbol.hasInstance]，在调用 instanceof 的时候，实际上构造函数会通过原型调用这个方法：

      class Fn {}
const fn = new Fn()
fn instanceof Fn             // true
// 等价于
Fn[Symbol.hasInstance](fn)   // true

      我们可以给构造函数自定义一个同名静态方法，从而遮蔽原型方法，自定义调用 insatnceof 时的返回值：

      class Fn{
    static [Symbol.hasInstance](){
        return false
    }
}
const fn = new Fn()
fn instanceof Fn            // false
Fn[Symbol.hasInstance](fn)  // false

    2. Symbol.toPrimitive：这个方法直接挂载在实例对象上，也即 obj[Symbol.toPrimitive]，每当当前上下文的对象必须以原始值表示的时候，都会首先尝试调用对象的这个方法。比如：

      //  hint 代表对象在当前上下文应该表示为哪种类型的原始值
function Fn(){
    this[Symbol.toPrimitive] = function(hint){
        switch(hint){
            case 'number': return 3;
            case 'string': return 'fn';
            case 'default': return 'default'    
        }
    }
}
const obj = new Fn()
// 这里的对象应该作为数字表示，所以 obj 转换为 3
obj - 3                 
// 这里的对象应该作为字符串表示，所以 obj 转换为 ‘fn’
alert(obj)              
// 这里的对象应该作为字符串表示，所以 obj 转换为 ‘fn’
String(obj)              
// 不知道是加法（3）还是字符串拼接（'fn'），所以 obj 转换为 'default'
obj + 3

    # Object 类型

    Object 除了通常所说的对象，还包括了函数和数组。

    # Object 属性访问

    以下方法定义或者修改属性：

    • Object.defineProperty(obj,'a',{})：定义或者修改一个属性。与直接 obj.a=1不同的是，属性默认不可枚举
    • Object.defineProperties(obj,{})：定义或者修改多个属性

    以下方法获取属性：

    • Object.keys()：获取所有可枚举的实例属性，无法获取符号属性（配套的有 Object.values 和 Object.entries）

    • for(key in obj)：获取所有可枚举的实例属性 + 原型属性，无法获取符号属性

    • Object.getOwnPropertyNames：获取所有的实例普通属性

    • Object.getOwnProertySymbols：获取所有的实例符号属性

    • Reflect.ownKeys()：获取所有的实例属性（无论是否可以枚举，无论是不是符号属性）

    以下方法获取属性的特性：

    • Object.getOwnPropertyDescriptor(obj,a)：获取某个实例属性的特性描述对象
    • Object.getOwnPropertyDescriptors(obj)：获取所有实例属性的特性描述对象

    以下方法判断是否存在属性：

    • obj.hasOwnProperty('a')：判断实例本身是否存在某个属性
    • a in obj：判断实例或者其原型上是否存在某个属性

    以下方法判断属性是否可枚举：

    • obj.propertyIsEnumerable('a')：判断实例属性是否可枚举

    # valueOf() 和 toString()

    valueOf() ：

    当通过基本类型的装箱对象（比如 new Number、new Boolean、new String）调用的时候，返回结果是对应的字符串、数值或者布尔值；当通过非装箱对象（比如对象字面量、new Object）调用的时候，返回结果是对象本身

    toString()：

    当通过基本类型的装箱对象调用的时候，返回结果是对应的字符串、数值或者布尔值的字符串表示；当通过非装箱对象调用的时候，则使用 Object 原型上的方法，返回结果是 "[object Object]"。

    **PS：**如果对象是数组，则使用 Array 原型上的方法，对于空数组返回 ""；对于非空数组，会针对每个元素调用一次 toString()，并将结果字符串通过逗号连接起来，比如 [1,2].toString() 返回 "1,2"。

    # Boolean 类型

    下面的值属于假值（falsy value），经过 Boolean 转型函数后得到 false：

    0   +0   -0   NaN   undefined   null   ""   false   document.all

    注意：

    • ""是假值，" "不是假值。后者本质上代表的是一个空格字符，它是有实体的
    • 空数组和空对象不是假值。它们虽然是“空的”，但是都是一个有效的对象，是有实体的
