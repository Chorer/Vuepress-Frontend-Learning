    ES next 新特性

    总结一些 ES6 之后新的语言特性

    # ES7（2016）

    # （1）Array.prototype.inlcudes()

    判断数组中是否包含某个元素

    # （2）指数操作符 **

    2**3 等同于以前的 Math.pow(2,3)

    # ES8（2017）

    # （1）async/await

    新的异步编码方式

    # （2）对象遍历

    Object.keys() 获取对象的所有 key，Object.values() 获取对象的所有 value，Object.entries() 获取对象的所有键值对

    # ES9（2018）

    # （1）异步迭代

    async/await + for...of，实现串行

    # （2）Promise.finally()

    不管 Promise 的状态是落定为成功还是失败，都会执行 finally 里面的回调函数

    # （3）展开运算符

    ... 用于合并或者展开对象和数组

    # ES10（2019）

    # （1）flat 和 flatMap

    flat 对数组进行扁平化，flatMap 对数组进行扁平化之后再 map 映射

    # （2）trimStart 和 trimEnd

    去除字符串前面或者后面的空格

    # ES11（2020）

    # （1）空值合并运算符

    逻辑或 || 的规则是，如果左操作数是假值，则返回右操作数；而空值合并运算符 ?? 的规则是，如果左操作数是 null 或者 undefined，则返回右操作数。

    这个区别很重要。为什么呢？看下面的例子：

    let a = 0
let b = a || 123          // b 最后是 123
let c = a ?? 123          // c 最后是 0

    我们期望的结果其实是，如果 a 是一个有效值，即使它是 0，也要让 b 等于 a。但使用 || 的时候，0 会被解析为假值，导致 b 最终等于 123；而如果使用 ??，则不存在这个问题，因为 0 不是 null 或者 undefined，所以 c 最终可以等于 0。

    # （2）可选链

    可选链 ?. 可以检测不确定的中间节点：

    let obj = {}
console.log(obj.a.b)    // obj.a 是 undefined，无法读取它的 b 属性，这里会报错
console.log(obj?.a?.b)  // 不会报错
console.log(obj.a?.b)   // 不会报错

    # （3）Promise.allSettled()

    Promise.all 只有所有的 Promise 状态都落定为成功，才会返回一个结果数组；而 Promise.allSettled 则不管 Promise 的状态是落定为成功还是失败，都会被记录在返回的数组中

    # （4）BigInt

    新的 JS 基本数据类型。在以前，JS 能够表示的最大整数是 2^53-1，一旦大于这个数，那么精度就会丢失，导致出现 2^53 === 2^53+1 这种情况。ES10 引入了 BigInt，它可以表示任意大的整数，通过 123n 或者 BigInt(123) 可以创建一个 BigInt。

    # （5）import()

    提供动态导入、按需导入的功能

    # ES12（2021）

    # （1）replaceAll

    符合规则的所有字符都会被替换掉

    # （2）Promsie.any

    Promise.any 和 Promise.all 是相反的。Promise.all 只要有一个失败，就返回失败的 promise，而 Promise.any 只要有一个成功，就返回成功的 promise

    # （3）WeakRef

    在以前，WeakMap 中的对象使用的是弱引用，而 WeakRef 提供了一个更加直接的方式去创建弱引用

    # （4）数字分隔符

    允许给数字字面量使用下划线作为分隔符，从而加强可读性：

    1_000_000_000 === 1000000000     // true
