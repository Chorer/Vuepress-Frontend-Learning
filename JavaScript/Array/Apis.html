    数组的常用方法

    vuePress-theme-reco    2021

    数组的常用方法

    # 会改变原数组的方法

    # 1）fill

    fill(replace,range1,range2)，用某个值替换数组中的全部值或者部分值。

    const arr = [0,0,0,0,0]
arr.fill(5)                 // [5,5,5,5,5]
arr.fill(5,2)               // [0,0,5,5,5]
arr.fill(5,2,4)             // [0,0,5,5,0] 
arr.fill(5,-3,-1)           // [0,0,5,5,0]  负值索引 => 负值索引 + 数组长度
arr.fill(5,-100,-90)        // 越界，无效
arr.fill(5,100,90)          // 越界，无效
arr.fill(5,4,2)             // 反向，无效

    # 2）copyWithin

    copyWithin(insert,range1,range2)，复制数组的某个部分，顶替数组中的某些元素。

    const arr = [0,1,2,3,4,5,6,7,8]
arr.copyWithin(4)                   // [0,1,2,3,0,1,2,3,4]   缺省范围为整个数组
arr.copyWithin(4,7)                 // [0,1,2,3,7,8,6,7,8]
arr.copyWithin(4,6,9)               // [0,1,2,3,6,7,8,7,8] 

// 对于负值索引、反向索引和越界索引的处理，和 fill 方法类似

    # 3）push 和 pop

    往数组末尾增加或者移除元素：

    const arr = [1,2,3]
arr.push(4,5)            // 返回最终的数组长度 5，数组变成 [1,2,3,4,5]
arr.pop()                // 返回移除的元素 5，数组变成 [1,2,3,4]

    # 4）shift 和 unshift

    往数组头部增加或者移除元素：

    const arr = [3,4,5]
arr.unshift(1,2)         // 返回最终的数组长度 5，数组变成 [1,2,3,4,5]
arr.shift()              // 返回移除的元素 1，数组变成 [2,3,4,5]

    # 5）reverse 和 sort

    reverse 将数组元素反向排列：

    [1,2,3].reverse()           // [3,2,1]

    显然这种排序方式不够灵活，于是有了 sort：

    const arr = [1,2,5,10]   
// 没有传入比较函数，会对每个元素调用 toString，在字符串层面进行比较，因此 "10"<"2"
arr.sort()                  // [1,10,2,5] 
// 传入比较函数，从小到大排序
arr.sort((a,b) => a<b?-1:a>b?1:0)     // [1,2,5,10]
arr.sort((a,b) => a-b)                // [1,2,5,10]
// 传入比较函数，从大到小排序
arr.sort((a,b) => a>b?-1:a<b?1:0)     // [10,5,2,1]

    比较函数作用的原理，如果要让最终排序结果中，a 出现在 b 的前面，那么就得返回 -1，a 出现在 b 的后面，就得返回 1，a 和 b 相等，就得返回 0。从小到大的排序中，如果 a<b，那么 a 必须出现在 b 的左边，而为了让 a 确实出现在 b 的左边，就得在 a<b 的时候返回 -1，其他同理。

    # 6）splice

    用于给数组删除、添加或者替换元素。第一个参数表示进行操作的位置，第二个参数表示删除多少个元素，第三个参数表示添加哪些新元素，调用之后返回一个数组，数组中包含被删除的元素（没有删除就是返回空数组）

    const arr = [1,2,3,4,5]

// 删除：在索引1这里操作，删除2个元素
arr.splice(1,2)                 // 返回 [2,3]，arr 变成 [1,4,5]

// 添加：在索引1这里操作，删除0个元素，添加2个元素（注意是插入到索引1前面，不是后面）
arr.splice(1,0,"a","b")          // 返回 []，arr 变成 [1,"a","b",2,3,4,5]

// 替换：删除+添加就是替换
arr.splice(1,2,"a","b")          // 返回 [1,"a","b",4,5]

    # 不会改变原数组的方法

    # 1）valueOf

    数组本质上属于引用类型中的 Object，因此就和对象字面量调用 valueOf 一样，数组调用 valueOf 也会返回自身：

    [1,2,3].valueOf()              // [1,2,3]

    # 2）toString

    会给数组中的每个元素调用 toString，之后再用逗号连接起来形成一个字符串：

    [1,2,3].toString()              // "1,2,3"
[{a:1},{b:2}].toString()        // "[obejct Object],[object Object]"

    # 3）join

    在 toString 的基础上，如果不希望连接符是逗号，可以用 join 自定义连接符：

    [1,2,3].join()                // "1,2,3"  缺省是逗号作为连接符
[1,2,3].join('.')             // "1.2.3"
[{},{},{}].join('**')         // "[object Object]**[object Object]**[object Object]"

    # 4）concat

    用于合并数组：

    const arr = [1,2,3]
// 接收多个参数，非数组参数直接加入，数组参数则拍平、取出每个元素加入
arr.concat("a",[5,6])            // [1,2,3,"a",5,6]
arr.concat("b",[[5],[6,7]])      // [1,2,3,"b",[5],[6,7]]

    通过设置数组的 Symbol.isConcatSpreadable 为 false，可以阻止数组被拍平，从而直接将数组加入：

    const arr1 = [1,2,3]
const arr2 = [4,5]
arr2[Symbol.isConcatSpreadable] = false
arr1.concat("a",arr2)               // [1,2,3,"a",[4,5]]

    通过设置类数组对象的 Symbol.isConcatSpreadable 为 true，可以让类数组对象被拍平，从而取出每个元素加入：

    const arr1 = [1,2,3]
const arr2 = document.querySelectorAll('div')
arr2[Symbol.isConcatSpreadable] = true
arr1.concat("a",arr2)         // [1,2,3,"a",div,div,div,...,div]

    # 5）slice

    数组切片，类似于字符串的 slice 方法：

    const arr = [1,2,3,4,5]
arr.slice(1)           // [2,3,4,5]
arr.slice(1,4)         // [2,3,4]
arr.slice(-4,-1)       // [2,3,4]     负值 => 数组长度加负值
arr.slice(4,1)         // []          反向索引，返回空数组

    # 6）indexOf 和 lastIndexOf 和 includes

    按严格相等去搜索数组元素：

    const arr = ['a','b','c','d','a','e']

arr.indexOf('b')           // 从前往后查找'b'，返回它的索引1
arr,indexOf('b',2)         // 从索引2开始，从前往后查找'b'，找不到，返回 -1

arr.lastIndexOf('a')       // 从后往前查找'a'，返回它的索引4
arr.lastIndexOf('a',2)     // 从索引2开始，从后往前查找'a'，返回它的索引0

arr.includes('c')          // 数组存在'c'，返回 true
arr.includes('c',3)        // 从索引3开始，数组不存在'c'，返回 false 
arr.includes('c',300)      // 超出数组长度，返回 false
arr.includes('c',-2)       // 负值=>负值+数组长度=>4，从索引4开始查找，返回 false
arr.includes('c',-100)     // 负值=>负值+数组长度=>-94，从头开始查找，返回 true

    为什么有了 indexOf 还要引入 inlcudes？一个是因为 includes 在语义表达上更加清晰，找不到元素就返回 false 而不是返回 -1，二是可以准确判断 NaN：

    [1,NaN].indexOf(NaN)     // -1
[1,NaN],includes(NaN)    // true

// 然而，inlcudes 仍然无法准确判断±0，会认为两者相等
[1,+0].includes(-0)      // true
[1,0].includes(+0)       // true

    # 7）flat

    用于数组扁平化（数组降维）。传入的参数代表对于数组中的每一个元素，要降维多少次，默认为 1 次：

    [1,[2,3],[[4,5],6]].flat()          // [1,2,3,[4,5],6]
[1,[2,3],[[4,5],6]].flat(2)          // [1,2,3,4,5,6]

    # 用于遍历的方法

    # 注意事项

    下面介绍的很多方法都可以接受两个参数，一个是回调函数，一个是 thisArg 绑定。注意这里的 thisArg 本质上是通过 call 绑定给回调函数的，而箭头函数并没有自己的 this，call 对它无效，因此如果要使用 thisArg，则这里的回调函数不能是箭头函数。

    # 1）find 和 findIndex

    通过indexOf 搜索数组，无法自定义搜索条件，所以改用 find。find 会对每个元素执行一次回调函数，直到找到符合条件的元素，就将这个元素返回（永远只返回一个），并结束函数执行；找不到则返回 undefined

    // 传参 item、index 和 arr，可以绑定 thisArg
['a','b','c'].find(function(item,index,arr){
    console.log(`${index}-${item}`,arr,this)
},{})
/* 
0-a  ['a','b','c']  {}
1-b  ['a','b','c']  {}
2-c  ['a','b','c']  {} 
*/
// 一旦找到第一个符合条件的元素，就将其返回
[1,2,3,4,5].find((item,index,arr) => item > 2)      // 3

    findIndex 和 find 一模一样，但是返回的是元素的索引：

    [1,2,3,4,5].findIndex((item,index) => item > 2)      // 2

    # 2）filter

    通过 find 搜索数组，只能找到一个符合条件的元素，而 filter 可以筛选出所有符合条件的元素：

    [1,2,3,4,5].filter((item,index) => item > 2)        // 返回 [3，4，5]
[1,2,3,4,5].filter((item,index) => item > 100)      // 没有符合的元素，返回 []

    # 3）forEach

    可以遍历每个元素，执行特定的操作。不会新建数组也不会修改原数组，总是返回 undefined（即使明确指定了返回值）

    // res 为 undefined
const res = ['a','b','c'].forEach(function(item,index,arr){
    console.log(`${index}-${item}`,arr,this)
    return 123
},{})                                                        
// 遍历过程中即使push了新元素，也仍然按照原数组长度进行遍历。打印 1，2
[1,2].forEach((item,index,arr) => {
    arr.push(3,4,5)
    console.log(item)
})
// 遍历过程中可以提前修改未遍历元素。打印 1，100
[1,2].forEach((item,index,arr) => {
    arr[1] = '100'
    console.log(item)
})
// 遍历过程中可以 return，但只是结束当前这次遍历，无法跳出整个 forEach。打印 2
[1,2].forEach((item,index,arr) => {
    if(index == 0) return
    console.log(item)
})
// 遍历过程中会自动跳过 empty 元素（null 和 undefined 不会跳过）。打印 1,2,4
[1,2,,4].forEach((item,index,arr) => {    
    console.log(item)
})

    # 4）map

    将原数组的每个元素映射为执行回调函数之后的返回值：

    // 返回新数组 [2,4,6,8]
[1,2,3,4].map((item,index) => item*2)

    # 5）some 和 every

    判断数组中是否部分元素或者所有元素都符合条件

    [1,2,3,4].some((item,index) => item>3)      // 至少有一个大于3的数，返回 true
[1,2,3,4].some((item,index) => item>100)    // 没有一个大于100的数，返回 false
[1,2,3,4].every((item,index) => item>0)     // 所有元素都大于0，返回 true
[1,2,3,4].every((item,index) => item>3)     // 并非所有元素都大于3，返回 false

    # 6）reduce 和 reduceRight

    reduce 可以归并数组的每个元素，最终构建一个累计归并值作为返回值。

    语法：arr.reduce((acc,cur,index,arr) => {...},baseAcc)

    它接受两个参数，一个是回调函数，一个是初始累计归并值。其中，回调的当前返回值会作为下次迭代所使用的 acc，也即累计归并值。初始累计归并值缺省是第一个非 empty 元素，且此时会从该元素的下一个元素开始迭代。

    // 没有提供初始累计归并值，因此缺省是1，并且从2开始迭代
[1,2,3,4].reduce((acc,cur) => acc + cur)              // 10

// 提供100作为初始累计归并值，从1开始迭代
[1,2,3,4].reduce((acc,cur) => acc + cur,100)          // 110

// 二维数组转化为一维数组，concat 本身会拍平一维数组
[1,2,[3,4]].reduce((acc,cur) => acc.concat(cur),[])       // [1,2,3,4]

    reduceRight 和 reduce 完全一样，只是从后往前遍历。

    # 7）keys 和 values 和 entries

    这三个方法都会返回一个数组迭代器对象：

    for(x of ['a','b','c'].keys()){
    console.log(x)             // 0 1 2
}
for(x of ['a','b','c'].values()){
    console.log(x)             // 'a'  'b'  'c'
}
for(x of ['a','b','c'].entries()){
    console.log(x)             // [0,'a']  [1,'b']  [2,'c']
}
